# Draft: Accessibility Testing and Tooling Workflow

A practical workflow for automated and manual accessibility testing.

## TLDR

- Automated checks catch only a fraction of issues
- Manual testing with keyboard and screen readers is essential
- CI integration prevents regressions at scale

## Outline

1. Test strategy: automated vs manual coverage
2. Keyboard navigation and focus management checks
3. Screen reader testing basics
4. Tooling: Axe, Lighthouse, WAVE, and linters
5. CI pipelines and regression prevention
6. Accessibility bug triage and prioritization
